# Agent workflow

이 저장소에서 자동화 에이전트도 사람과 같은 티켓 게이트를 따른다.

1. `docs/BACKLOG.md`와 관련 제품·아키텍처 계약을 먼저 읽는다.
2. Ready 티켓 한 개의 범위 안에서 작업하고 ID를 변경 설명에 남긴다.
3. 구현 전에 상태·수식·저장 영향과 수용 기준을 확인한다.
4. 게임 규칙은 `src/game`, 브라우저 생명주기는 `src/hooks`, 표시는 `src/components`에 둔다.
5. 밸런스·저장 계약을 바꾸면 문서와 회귀 테스트를 같은 변경에서 갱신한다.
6. Review → Verify → Test를 생략하지 않는다.
7. `npm run verify` 통과와 필요한 수동 증거 없이 Done으로 표시하지 않는다.

보상 복제, 저장 손실, 진행 차단 가능성이 발견되면 신규 기능보다 해당 결함을 우선한다. 서버·결제·소셜 범위는 제품 승인 티켓 없이 추론해 추가하지 않는다.
