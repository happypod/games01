# 스킬 정리

이 문서는 제작에 필요한 역량과 게임 안에서 플레이어가 사용하는 스킬을 함께 정의한다. 티켓의 `Skill tags`는 제작 역량을, `Game skill`은 콘텐츠 ID를 뜻한다.

## 1. 제작 역량 태그

숙련도는 `L1 이해`, `L2 가이드 하 수행`, `L3 독립 수행`, `L4 설계·리뷰`로 정의한다.

| 태그 | 역량 | 최소 | 주요 책임 |
|---|---|---:|---|
| PROD-LOOP | 핵심 루프·범위 관리 | L3 | 제품 계약, 비범위, 지표, 우선순위 |
| GD-BAL | 성장 경제·밸런스 시뮬레이션 | L4 | 수식, 비용·보상 곡선, 환생 시간 |
| GD-SKILL | 전투·스킬 효과 설계 | L3 | 트리거, 쿨다운, 중첩, 강화 규칙 |
| ENG-SIM | 결정론적 시간·전투 시뮬레이션 | L4 | `advanceGame`, 불변식, 장시간 계산 |
| ENG-STATE | 상태·명령·이벤트 모델링 | L3 | GameState, 원자적 명령, 파생 상태 |
| ENG-DATA | 데이터 기반 콘텐츠 | L3 | 적·스킬·강화 설정과 검증 |
| ENG-SAVE | 저장·마이그레이션·복구 | L3 | 버전, A/B 슬롯, 오프라인 중복 방지 |
| FE-GAME | React 게임 UI·런타임 연결 | L3 | 화면, 상태 구독, 브라우저 생명주기 |
| UX-FEEDBACK | 정보 구조·피드백·접근성 | L3 | 전투 인지, 잠금·오류·최대 상태 |
| QA-DOMAIN | 경계값·결정론·불변식 테스트 | L3 | 엔진과 경제 회귀 자동화 |
| QA-E2E | 브라우저 회귀·탐색 테스트 | L3 | 저장·재접속·오프라인 전체 흐름 |
| REL-CI | 빌드·CI·릴리스 게이트 | L2 | 재현 가능한 설치와 품질 명령 |
| PLAYTEST | 플레이 로그·튜닝 판단 | L2 | 목표 시간, 선택 빈도, 체감 분석 |
| ART-DIR | 시각 방향·자산 계약 | L3 | 스타일 일관성, asset ID, 규격·권리·fallback, 성능 예산 |
| ART-2D | 2D 게임 일러스트 제작 | L3 | 캐릭터·환경·카드·결과 화면 제작과 웹 최적화 |

최소 팀은 시스템 디자이너/PO, 코어·프런트엔드 엔지니어, 파트타임 UX·QA 역할로 구성할 수 있다. 한 사람이 여러 역할을 맡더라도 `ENG-SIM` 구현과 `QA-DOMAIN` 검토는 가능하면 분리한다.

## 2. 현재 게임 스킬

| ID | 이름 | 종류·해금 | 현재 효과 | 강화 |
|---|---|---|---|---|
| `powerStrike` | 화염 강타 | 자동 액티브, Lv.1 | 준비 시 기본 공격 대신 250% 피해, 쿨다운 5초 | 랭크당 피해 배율 +25%p, 최대 10 |
| `ironWill` | 강철 의지 | 패시브, Lv.3 | 최대 HP·방어력 +10% | 랭크당 +10%, 최대 10 |
| `fortune` | 전리품 감각 | 패시브, Lv.5 | 골드 획득 +12% | 랭크당 +12%, 최대 10 |

- 레벨업마다 스킬 포인트 1점을 얻는다.
- 랭크 0~3은 1포인트, 4~7은 2포인트, 8~9는 3포인트가 든다.
- 패시브는 저장된 스탯에 더하지 않고 원본 상태에서 매번 파생해 중복 누적을 막는다.
- 액티브 쿨다운은 밀리초로 저장되어 새로고침과 오프라인 진행에서도 유지된다.

## 3. 다음 스킬 팩

시드 RNG와 효과 런타임이 준비된 뒤 다음 순서로 확장한다.

| 후보 ID | 이름 | 트리거 | 효과 | 선행 티켓 |
|---|---|---|---|---|
| `emergencyMend` | 응급 재생 | HP 40% 이하 | 최대 HP 20% 회복, 25초 쿨다운 | IRPG-105 |
| `warCry` | 전투 함성 | 자동 우선순위 | 8초간 공격력 +20%, 20초 쿨다운 | IRPG-105 |
| `emberShield` | 잿불 방벽 | 전투 시작 | 최대 HP 15% 보호막 | IRPG-105 |

효과 런타임은 `priority → skillId` 순으로 결정론적으로 평가한다. 같은 버프는 중첩하지 않고 지속시간을 갱신하며, 회복 > 방어 > 버프 > 공격 순을 기본 우선순위로 사용한다.

## 4. 스킬 시스템 완료 조건

- 동일 상태·시간·시드에서 시전 순서와 최종 상태가 같다.
- 쿨다운과 지속시간은 음수가 되지 않는다.
- 패시브는 저장·로드·재계산 후 중복 적용되지 않는다.
- 강화 실패 시 포인트와 랭크가 모두 유지된다.
- 잠금·비용·현재 효과·다음 효과·최대 랭크가 UI에 표시된다.
- 온라인과 오프라인 계산이 같은 규칙을 사용한다.
