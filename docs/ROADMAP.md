# 전체 로드맵

## 운영 원칙

기능 수가 아니라 아래 사용자 흐름의 완결성을 기준으로 릴리스한다.

```text
신규 게임 → 자동 전투 → 보상 → 강화 → 보스 → 저장
          → 재접속 → 오프라인 결과 → 환생 → 더 빠른 재진행
```

각 단계는 `설계 계약 → 구현 → 정적 검증 → 자동 테스트 → 플레이 검증`을 모두 통과해야 종료된다.

## 단계와 게이트

| 단계 | 상태 | 목표 | 종료 조건 |
|---|---|---|---|
| G0 제품 계약 | 완료 | 루프, 범위, 수식, 상태, 비범위 고정 | 모든 P0 기능에 수용 기준과 소유 스킬 태그가 있다. |
| G1 헤드리스 엔진 | 완료 | UI 없이 전투·보상·성장 실행 | 같은 상태·시간의 결과가 같고 핵심 숫자 불변식 테스트가 통과한다. |
| G2 플레이 수직 조각 | 완료 | 화면에서 전투→강화→스킬→환생 수행 | 반응형 첫 화면에서 핵심 명령을 키보드로 실행할 수 있다. |
| G3 로컬 지속성 | 완료 | 저장·복구·오프라인 진행 | 새로고침, 손상 저장, 시계 역행, 8시간 상한을 안전하게 처리한다. |
| G4 알파 하드닝 | 완료 | 저장 안전성, E2E, 접근성, 디버그 도구 | A/B 저장과 마이그레이션, 브라우저 E2E, 24시간 soak가 통과한다. |
| G5 밸런스 알파 | 계획 | 30~45분 첫 환생과 재진행 가속 검증 | 자동 10회 수치 검증과 별도의 외부 사용자 플레이테스트에서 진행 차단이 없고 목표 시간이 범위 안에 든다. |
| G6 콘텐츠 베타 | 계획 | 콘텐츠 다양성과 장기 목표 | 지역 3개, 스킬 6종, 업적·퀘스트가 데이터만으로 추가된다. |
| G7 온라인 전환 | 보류 | 계정·클라우드·운영 | 서버 권위 시간, 인증, 보상 멱등성, 개인정보·운영 정책이 승인된다. |

### 이후 — 콘텐츠 베타

캐릭터·맵·카드·이벤트·승패 화면의 상세 의존 순서는 [다음 콘텐츠·시각 로드맵](NEXT_CONTENT_ROADMAP.md)을 따른다.

- 지역별 적·보스 패턴과 자동 스킬 3종 추가
- 장비 슬롯은 고정 강화 트랙 검증 후에만 랜덤 드롭으로 확장
- 퀘스트·업적은 기존 행동 이벤트를 소비하는 방식으로 추가
- IndexedDB 저장 어댑터와 여러 저장 슬롯 검토
- PWA는 오프라인 실행이 제품 요구가 된 뒤 추가

### 온라인 전환 조건

다음 요구 중 하나가 확정될 때만 서버 작업을 시작한다.

- 결제 또는 유료 재화
- 경쟁 랭킹·PvP·길드
- 여러 기기 동기화
- 라이브 이벤트와 서버 보상

이때 클라이언트 시계와 재화 계산을 신뢰하지 않고, 서버 권위 시간·명령 검증·멱등 보상 키를 별도 설계한다.

## 핵심 지표

| 지표 | 초기 목표 |
|---|---:|
| 첫 전투 인지 | 진입 후 3초 이내 |
| 일반 적 처치 시간 | 2~6초 |
| 첫 의미 있는 강화 | 2분 이내 |
| 첫 보스 진입 | 30~60초 |
| 첫 보스 격파 | 1~3분 |
| 첫 환생 관문 | 30~45분 |
| 첫 환생 후 최고점 재도달 | 이전 시간의 50~70% |
| 10분 내 성장 선택 | 3회 이상 |
| 데이터 손실·보상 중복·진행 차단 | 0건 |

IRPG-204 자동 밸런스 검증은 고정 seed·전략 10개에서 첫 환생 관문 중앙값 33분 04.5초, 범위 30분 48초~37분 03초를 기록한다. 이는 외부 사용자의 이해·기대·재미를 증명하지 않으며 IRPG-205 실제 사용자 플레이테스트에서 별도로 검증한다. 첫 환생 후 최고점 재도달은 IRPG-206에서 보정한다.

## 주요 위험

| 위험 | 대응 |
|---|---|
| 브라우저 백그라운드 타이머 제한 | 실제 경과 시간과 나머지 밀리초를 상태에 보존 |
| 사용자 시계 조작 | 로컬 알파는 상한·역행만 방어, 온라인 전환 시 서버 권위 시간 |
| 지수 성장 폭주 | 최대 스테이지, 안전 정수 상한, 장시간 불변식 테스트 |
| 저장 손상·구버전 | decoder, A/B revision, 검증 후 migration, 미래 포맷 쓰기 차단 |
| 환생이 손해처럼 보임 | 예상 보상과 유지·초기화 항목 표시, 전후 시간 지표 측정 |
| 범위 팽창 | 비범위 기능은 P0 게이트 통과 전 착수 금지 |
